# myLedger 1.0 业务归档

本文档描述 **1.0 当前已交付** 的产品边界、数据域与运维要点，便于交接与长期查阅。接口明细见 [api-http.md](api-http.md)。

---

## 1. 产品范围

个人记账：**用户** → **资金账户**（现金/卡/第三方支付等）→ **收支分类** → **流水**；提供概览、流水、记一笔、**统计**（月 × 分类矩阵，可按账户筛选）、**我的**（资料、资金账户、分类、改密）。

- **无「多账本」维度**：数据按用户归属，通过资金账户区分资金口径。
- **客户端**：当前为 **H5**（`myledger-h5`）；后端 API（`myledger-api`）可供后续其它端复用。

---

## 2. 核心业务对象

| 对象 | 说明 |
|------|------|
| 用户 | 注册/登录；`user_id` 由服务端 Session 注入业务 SQL，请求体不传 `user_id`。 |
| 资金账户 | 每用户多账户；其一为默认；流水必须关联 `account_id`。 |
| 分类 | 每用户、分收入/支出（`income` / `expense`）的一级分类。 |
| 流水 | 金额恒正；`entry_type` 与分类 `type` 一致；`remark` 在应用层要求非空（见接口文档）。 |
| 报表 | 按月汇总、按分类、按日（接口保留；首页是否展示按产品迭代）、**月区间 × 分类矩阵**（`matrixByMonthCategory`，月份字段为 `bill_month`）。 |

---

## 3. 数据库（MySQL）

表前缀 **`ml_`**：`ml_user`、`ml_account`、`ml_category`、`ml_entry`。时间字段 **`created_at` / `updated_at`** 由应用写入（无库端默认值，兼容旧版 MySQL）。

全量建表脚本：**[sql/init_schema.sql](sql/init_schema.sql)**。先在 MySQL 中建好目标库并授权，将脚本内 **`USE …`** 改为该库名，并与 **`myledger-api`** 数据源 URL 中的库名一致。

密码当前为 MySQL **`SHA2(明文, 256)`**，仅适合可控环境；生产应替换为更强方案。

---

## 4. 接口与鉴权（摘要）

- **健康检查**：`GET /api/health`（不经 dbfound）。
- **登录**：`POST /api/auth/login`，Cookie Session；业务请求 `withCredentials`。
- **dbfound**：路径与 `resources/model/**/*.xml` 对应；参数 **snake_case**；匿名白名单见 [api-http.md](api-http.md)。

完整路径与字段表：**[api-http.md](api-http.md)**。

---

## 5. 本地运行（摘要）

1. 按上文执行 SQL，配置 `myledger-api` 数据源与 Spring profile（开发默认 `dev`，生产使用非 `dev`）。
2. 后端：`cd myledger-api && mvn spring-boot:run`（默认端口 **8080**）。
3. 前端：`cd myledger-h5 && npm install && npm run dev`（默认 **5173**；`Node` 版本以 `package.json` / `.nvmrc` 为准）。

开发期 **`VITE_API_BASE` 留空** 可走 Vite 代理，避免跨域。生产构建配置 `VITE_API_BASE`。CORS 开发较宽松，**上线须收紧**。

更细的目录说明、Maven profile、局域网调试与常见安装问题见仓库根目录 **[README.md](../README.md)**。
