# 版本说明

重大变更（数据模型、对外 API、部署方式、业务规则反转）请在此 **追加** 简短记录：日期、类型（技术/业务）、一句话摘要、必要时的迁移提示。日常小改可不写。

## 1.0（当前）

- **2026-04-21（技术）**：登录不再按用户清空 **`ml_refresh_token`**，允许同一账号多端并存 refresh；**`/api/auth/logout`** 与 **refresh 轮换** 仍只影响当前这一条令牌。
- **2026-04-21（技术）**：API 鉴权改为 **JWT**（access + refresh，refresh 存 **`ml_refresh_token`** 哈希）；**`BearerAuthFilter`** 校验 `Authorization: Bearer` 后写入 **request attribute**，dbfound **`scope="request"`** 读取 `user_id`。已有库需补 **`ml_refresh_token`** 表（见 `docs/sql/init_schema.sql`）；生产用环境变量覆盖 **`myledger.security.jwt.secret`**。H5：`localStorage` 存令牌，`http` **`withCredentials: false`**。
- **2026-04-21（技术）**：后端数据访问**统一 dbfound**，已移除 **`JdbcTemplate`** 与仅为模板服务的 **`spring.datasource`**；refresh 与用户按 id 加载见 **`model/auth/refresh_token.xml`**、**`user/user.query!findByIdForAuth`**。约定写入 **`.cursor/rules/dbfound-framework.mdc`**。
- **范围**：用户、资金账户、收支分类、流水；统计含月汇总、按分类、月×分类矩阵、按日（接口）；H5 交付。
- **数据**：`ml_user`、`ml_account`、`ml_category`、`ml_entry`、`ml_refresh_token`；无 `ml_ledger`。
- **文档**：业务总览见 [业务归档.md](业务归档.md)；HTTP 路径见 [api-http.md](api-http.md)。
